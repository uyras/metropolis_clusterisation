Программа для поиска кластеров в конфигурации магнитной системы
=====
У нас есть файл с системой mfsys и файл с конфигурациями.

Файл с конфигурациями определяется следующим образом. 
В каждой строке файла записано по одной конфигурации. Это строка из N символов, каждый символ это 0 либо 1. 0 - значит что в исходном файле направление спина не изменено, 1 - что спин перевернут.

Программа определяет соседние спины (с учетом ПГУ и текущей конфигурации), и если их энергия <0, то пара спинов считается принадлежащей одному кластеру.

Для каждой строки (для каждой конфигурации) программа считает размер минимального кластера, максимального, количество, средний размер. Результат для каждой строки выдается на экран.

## Компиляция

```bash
make build && cd build
cmake ..
cmake --build .
```

## Запуск
Формат запуска программы: `./metropolis_clusterisation <sys.mfsys> <state file.txt> <distance> <size_x> <size_y> <size_z>`, где:

* `<sys.mfsys>` - файл системы
* `<state file.txt>` - файл с состояниями
* `<distance>` - расстояние, на котором учитывать соседние спины
* `<size_x> <size_y> <size_z>` - размеры системы для ПГУ


В папке `example` есть пример. Запускать так (из корневой директории проекта):

```bash
build/./metropolis_clusterisation example/system.mfsys example/states.txt 0.8 20 20 0
```
